Rust Memory Visualizer

What This Does

Paste any Rust code and see exactly where each piece of data lives in memory: stack, heap, .rodata, .data, .bss, or .text.

Keyboard Shortcuts

Ctrl+EnterAnalyze code
EscClose modals
← →Step through timeline
SpacePlay/pause timeline (when focused)

How It Works

The analyzer tokenizes your Rust code and applies memory classification rules based on type patterns, declaration context, and Rust's ownership model. It does not compile your code — it uses heuristic analysis.

Memory Segments

  • Stack — local variables, function parameters, references, struct/enum values
  • Heap — Box, Vec, String, HashMap, Rc, Arc, and other dynamically-sized data
  • .rodata — string literals, const values, immutable statics, vtables
  • .data — mutable statics (static mut, interior mutability globals)
  • .bss — zero-initialized statics (no binary storage cost)
  • .text — function bodies, method implementations, closure code

Interactions

  • Hover over code to highlight its memory location
  • Hover over memory blocks to highlight source code
  • Click memory blocks for detailed information
  • Use the timeline to step through execution order
